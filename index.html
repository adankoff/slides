VERSION CONTROL FUNDAMENTALS

by Alex Dankoff

What is revision control?

  • A tool for tracking changes.
  • It's an Industry standard.
  • All open source code project’s are kept in source control.

Revision control in everyday life

  • We use revision control in all of our digital documents. Anything you can “Ctrl-z” is technically under revision control.
  • Wordpress publishing and editing is a type of version control.
  • Wordpress core code is kept in version control
  • Most open source plugins and themes are kept in version control

Why version control?

  • Tracking changes to our code: revision tracking.
  • Working in a team: merging and branching.
  • Deploying changes to environments: hooks.
  • Sharing your code from anywhere: online repositories

The different flavors of version control

Git - SVN - Mercucial

For a full comparison see:
timedoctor.com: git mecurial and cvs/svn comparison

Why I use git:

  • Most of the open source projects I code with are hosted on github
  • Most of the places I've worked use git.
  • It seems to be the one the community has gotten behind.
  • It has many options for free SAAS hosted repositories.

Online hosted git services

Github and Bitbucket

Both have great web interfaces, fast upload and download times, ease of use and external plugins.

Github

  • Unlimited free public repositories.
  • Paid private repositories.
  • Unlimited collaborators.

Good for public repositories or open source projects that need many collaborators.

Bitbucket

  • Unlimited free public repositories.
  • Unlimited free private repositories.
  • Paid collaborators over 5 users.

Good for private projects that will not be open source that only require a few collaborators.

GUI Tools for git

  • for Mac: SourceTree - More
  • for Windows: TortoiseGIT - More
  • Lot of IDE plugins also for sublime, phpstorm and more. Same commands, easier interface.

Installing git

Go to the git website and install depending on your different OS. There's an easy installer for both Windows and Mac.

Creating account and generating keys


ls -al ~/.ssh
ssh-keygen -t rsa -C "your_email@example.com"
ssh-add ~/.ssh/id_rsa
pbcopy < ~/.ssh/id_rsa.pub
ssh -T git@github.com

Creating, cloning and configuring a repo


git clone https://github.com/adankoff/wp-composer.git
git config --global user.name "Alex Dankoff"
git config --global user.email "a.dankoff@gmail.com"

.gitignore


*.log
.htaccess
sitemap.xml
wp-config.php
wp-content/uploads/

Adding files to the repo


git status #get current repo status
git add filename.txt #add specific file to my next commit
git status #get current repo status

git add --all #add all changes

Commiting files to the repo


git commit -m "added new files" #commit new files with message
git status #get current repo status

git commit -a -m "added new files" #add all and commit at the same time

Pushing files to the repo


git push origin master #push origin to master
git status #get current repo status
git diff #see differences made from presiouve

Branching from master


git branch temporary #creates branch temporary
git branch #lists branches
git checkout temporary #change branch to temporary

git checkout -b temporary #create branch and checkout branch

Pulling from master


git commit -m "add these files"
git status

Merging into master


git push origin master
git status
